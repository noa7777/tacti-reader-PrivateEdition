# 文档目录（TOC）交互需求整理

## 1. 核心需求

### 1.1 按钮高亮规则

| 场景 | 高亮状态 | 说明 |
|------|---------|------|
| 新打开的书籍 | 无任何按钮高亮 | 首次使用目录功能 |
| 使用过目录跳转过的书 | 保留上次选中/跳转的按钮高亮 | 每次打开目录时恢复 |
| 点击某个按钮 | 该按钮立即高亮 | 无论是叶子节点还是父节点 |
| 从深层返回浅层 | 返回路径上的按钮保持高亮 | 见"层级导航" |

### 1.2 按钮点击行为

| 节点类型 | 点击行为 | 界面变化 |
|---------|---------|---------|
| **有子节点的按钮** | 展开子节点（不跳转） | 显示下一层级的子按钮 |
| **无子节点的按钮（叶子）** | 跳转到对应页码 | 关闭对话框，跳转到目标页 |
| **已展开的父节点再次点击** | 保持选中高亮 + 展开子节点 | 子节点继续显示 |

### 1.3 层级导航（核心逻辑）

#### 示例场景
```
假设目录结构：
├── 第1层
│   ├── 第一章 (A) ── 第2层
│   │   ├── 第一节 (A1) ── 第3层
│   │   │   ├── 1.1 (A11) [叶子]
│   │   │   └── 1.2 (A12) [叶子]
│   │   └── 第二节 (A2) ── 第3层
│   │       ├── 2.1 (A21) [叶子]
│   │       └── 2.2 (A22) [叶子] ── 第4层
│   │           └── 2.2.1 (A221) [叶子]
│   └── 第二章 (B) ── 第2层
│       └── 第一节 (B1) [叶子]
```

```

**步骤5**（关键）：回到第2层点击"第一节"（有子节点）
→ 只显示1~3层，"第一节"高亮，其子节点展开

```
┌─────────────────────────────┐
│ 第1层                        │
│ [★第一章]  [第二章]          │

- 第4层消失（因为当前只展开到第3层）
- 第1层的"第一章"保持高亮（它是祖先节点）
- 第2层的"第一节"变为高亮（当前选中）
- "第二节"取消高亮（不再是选中路径）

**步骤6**：点击"1.1"（叶子节点）→ 跳转到对应页码，关闭对话框

## 2. 详细需求规格

### 2.1 显示层级计算

```python
def calculate_display_depths(selected_node):
    """
    计算需要显示的层级范围
    
    参数:
        selected_node: 当前选中的节点（可以是 None）
    
    返回:
        display_range: (min_depth, max_depth)
        ancestor_chain: [node1, node2, ..., selected_node]  # 从根到选中节点的链路
    """
    if not selected_node:
        return (1, 1), []
    
    # 找到从根到选中节点的完整链路
    chain = find_ancestor_chain(selected_node)
    max_display = len(chain)  # 选中节点所在深度
    
    # 如果选中节点有子节点，多显示一层
    if not selected_node.is_leaf:
        max_display += 1
    
    return (1, max_display), chain
```

### 2.2 高亮规则

```python
def get_highlighted_nodes(selected_node, ancestor_chain):
    """
    获取需要高亮的节点集合
    
    规则:
    1. 选中节点本身高亮
    2. 所有祖先节点高亮
    3. 同级兄弟节点不高亮
    """
    highlighted = set()
    
    if selected_node:
        # 选中节点
        highlighted.add(selected_node.node_id)
        
        # 所有祖先节点
        for node in ancestor_chain:
            highlighted.add(node.node_id)
    
    return highlighted
```

### 2.3 持久化存储

```python
class TocState:
    """TOC 对话框状态持久化"""
    
    def __init__(self):
        self.selected_id: str = ""      # 最后选中的节点 ID
        self.last_jump_page: int = -1    # 最后跳转到的页码（用于判断是否是叶子节点）
    
    def to_dict(self) -> dict:
        """序列化为字典，保存到配置文件"""
        return {
            "selected_id": self.selected_id,
            "last_jump_page": self.last_jump_page,
        }
    
    @classmethod
    def from_dict(cls, data: dict) -> 'TocState':
        """从字典反序列化"""
        state = cls()
        state.selected_id = data.get("selected_id", "")
        state.last_jump_page = data.get("last_jump_page", -1)
        return state
```

### 2.4 打开时的行为

```python
def on_open_toc_dialog():
    """
    打开 TOC 对话框时的行为
    """
    # 1. 加载保存的状态
    toc_state = load_toc_state(pdf_path)
    
    # 2. 如果是新书（无保存状态），显示第1层，无高亮
    if toc_state.selected_id == "":
        dialog = TocDialog(
            pdf_toc=pdf_toc,
            selected_id="",     # 无选中
            max_visible_depth=1  # 只显示第1层
        )
    
    # 3. 如果是旧书（有保存状态），恢复之前的选中状态
    else:
        dialog = TocDialog(
            pdf_toc=pdf_toc,
            selected_id=toc_state.selected_id,
            max_visible_depth=calculate_depth_from_node(toc_state.selected_id)
        )
```

### 2.5 关闭时的行为

```python
def on_close_toc_dialog(dialog):
    """
    关闭 TOC 对话框时的行为
    """
    # 1. 如果是通过叶子节点跳转关闭的
    if dialog.closed_by_leaf_click:
        save_toc_state(
            pdf_path=pdf_path,
            selected_id=dialog.selected_id,
            last_jump_page=dialog.jump_page
        )
    
    # 2. 如果是通过关闭按钮关闭的（未跳转）
    else:
        # 不保存状态（或保存但不更新 last_jump_page）
        pass
```

## 4. 边界情况处理

### 4.1 特殊场景

| 场景 | 处理方式 |
|------|---------|
| PDF 无目录 | 显示提示文字："该 PDF 没有目录" |
| 目录只有一层 | 直接显示所有按钮，点击即跳转 |
| 目录层级很深（>10层） | 正常支持，滚动查看 |
| 选中节点被删除（PDF 更新后） | 自动清除选中状态，显示第1层 |
| 快速连续点击不同层 | 每次点击都重新渲染，响应迅速 |

### 4.2 性能优化

- **懒加载**：只在展开时渲染子节点
- **缓存**：缓存树结构和祖先链路计算结果
- **虚拟滚动**：如果某层节点过多（>50个），考虑分页或搜索过滤

通过阅读现有 `TocDialog` 代码（[tactireader.py:669-948](file:///C:/Users/Administrator/Desktop/ml/tacti-reader-master/tacti-reader-master/tactireader.py#L669-L948)），发现以下问题：

| **按钮样式硬编码** | 未选中按钮颜色写死为 #F0F0F0 | 与主题不一致 |

## 7. 总结

### 核心原则

1. **选中即高亮**：无论点击哪个按钮，它都应立即高亮
2. **层级跟随**：显示范围严格跟随选中节点的深度
3. **祖先链高亮**：从根到选中节点的整条路径都要高亮
4. **状态持久**：每次打开目录都恢复上次的选中状态
5. **叶子即跳转**：叶子节点点击后立即跳转并关闭

### 优先级

- **P0（必须实现）**：基本的高亮、层级导航、叶子跳转
- **P1（重要）**：状态持久化、祖先链高亮
- **P2（优化）**：性能优化、边界情况处理
